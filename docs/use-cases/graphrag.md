# GraphRAG — Use Cases

> **Status**: 📋 Planned | **Priority**: P3 | **Issue**: [#12](https://github.com/sou350121/Pulsar/issues/12)

Builds a knowledge graph over Pulsar's signal archive — linking papers, methods, labs, and concepts — enabling path-based and cluster-based queries that flat vector search cannot answer.

---

## Use Case 1: Topic Cluster Discovery via Graph Traversal

**Scenario**: The researcher suspects that tactile sensing, foundation models, and sim-to-real transfer are converging into a single research thread but cannot verify this from individual paper summaries. They want to see the connecting path in Pulsar's archive.

**What happens**: GraphRAG represents each signal as a node with typed edges (cites, uses-method, shares-author, shares-dataset, related-to-concept). A path query between two or more concept nodes traverses the graph and returns the shortest connecting chain of papers and concepts. Multiple chains indicate a forming cluster.

**Example**:
```
Query: path("tactile sensing" → "foundation models" → "sim-to-real")

Path 1 (length 3):
  [tactile sensing] → TactileVLA (⚡, 2026-01-08)
                    → uses-dataset: DexGraspBench (synthetic tactile)
                    → DexGraspBench used-by: SimToReal-FM (⚡, 2026-01-22)
                    → [sim-to-real] + [foundation models]

Path 2 (length 2):
  [tactile sensing] → ViTac (🔧, 2025-12-14)
                    → cites: RT-2 (foundation model)
                    → [foundation models] + [sim-to-real via IsaacLab experiment]

Cluster confidence: HIGH (2 independent paths < length 4)
```

## Use Case 2: Emerging Subfield Detection

**Scenario**: The researcher wants Pulsar to automatically alert them when 5 or more papers with overlapping concepts appear within a 30-day window — a signal that a new subfield is crystallising before it gets a standard name.

**What happens**: A weekly graph analysis job computes connected components in the signal graph restricted to the past 30 days. Any component with 5 or more ⚡/🔧 nodes that shares at least 2 concept edges (method, dataset, or task) is flagged as an "emerging cluster". The cluster is assigned a provisional label (auto-generated by qwen3.5-plus from the node titles) and reported in the Sunday quality review.

**Example**:
```
[EMERGING CLUSTER — week of 2026-02-23]
Provisional label: "Generalised Dexterous Grasping with Synthetic Tactile Data"
Node count: 7 (5x ⚡, 2x 🔧)
Shared concepts: IsaacLab (6/7 nodes), GelSight simulation (5/7), in-hand rotation task (7/7)
Oldest node: 2026-01-31 | Newest: 2026-02-25
Recommended action: Add as candidate assumption to assumptions.json
```

## Use Case 3: Central Paper Identification

**Scenario**: Within the "robot learning from human video" cluster, the researcher wants to know which single paper is most influential — not by citation count (too slow to accumulate on preprints) but by graph centrality within Pulsar's archive.

**What happens**: GraphRAG computes PageRank or betweenness centrality over the signal graph for a specified concept cluster. The paper with the highest centrality score is the "hub paper" — the one most papers in the cluster either cite, share methods with, or share authors with. This identifies seminal work faster than waiting for Google Scholar citation counts to settle.

**Example**:
```
Cluster: "robot learning from human video"
Hub paper: "GROOT: Learning Generalizable Manipulation Policies with Object-Centric 3D Representations"
  Centrality score: 0.91
  Connections: cited-by 6 nodes, shares-method (object-centric repr.) with 4 nodes,
               shares-author (Yuke Zhu) with 3 nodes
  Rating: ⚡ (2025-11-04)
  Interpretation: Any new paper in this cluster that ignores GROOT likely has a gap.
```

## Use Case 4: Knowledge Evolution Timeline

**Scenario**: The researcher wants a chronological view of how "robot learning from demonstration" has evolved in Pulsar's tracked months — which methods appeared first, which were superseded, and which are still active.

**What happens**: GraphRAG generates a timeline graph by ordering nodes chronologically and drawing "supersedes" or "extends" edges inferred from citation and method overlap. The output is a text-based timeline with method generations annotated, suitable for pasting into a research memo or biweekly report.

**Example**:
```
Topic: "robot learning from demonstration" — evolution 2025-10 to 2026-02

2025-10: Behaviour Cloning baseline (📖) — ACT variants dominate benchmark
2025-11: Diffusion Policy extensions (🔧 x3) — faster sampling, RL fine-tuning
2025-12: VLA integration (⚡ x2) — language-conditioned imitation, Octo-2 released
2026-01: Synthetic data + foundation model pretraining (⚡ x4) — paradigm shift
2026-02: Real-robot scaling (⚡ x3) — pi0.5, PhysicalAI dataset, 1000-demo regimes

Active front (latest signals): real-robot data scaling + foundation model pretraining
Superseded: pure BC without diffusion or VLA conditioning (no new ⚡ since 2025-11)
```
